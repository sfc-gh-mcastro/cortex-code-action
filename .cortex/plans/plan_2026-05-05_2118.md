## Context

The current action only supports "tag mode" — it requires `@cortex-code` to appear in a comment body. For a PR review workflow triggered on `pull_request` events (opened/synchronize), we need **agent mode**: the action receives a `prompt` input directly and skips trigger detection.

The context parser already handles `pull_request` events, so the changes are minimal.

## Implementation Steps

### Step 1: Add `prompt` input to `action.yml`
Add a new `prompt` input that, when provided, puts the action into "agent mode" — bypassing trigger phrase detection and using the prompt directly.

```yaml
prompt:
  description: "Direct prompt for agent mode. When provided, skips trigger detection and uses this as the instruction."
  required: false
  default: ""
```

Pass it as `INPUT_PROMPT` env var in the run step.

### Step 2: Add agent mode to `src/entrypoints/run.ts`
After parsing context, detect mode:
- If `INPUT_PROMPT` is set and non-empty → **agent mode** (skip trigger check, skip human actor check, use prompt directly)
- Otherwise → **tag mode** (existing behavior)

In agent mode:
- Skip `containsTrigger()` and `isHumanActor()` checks
- Still set up auth, fetch PR data for context, and create tracking comment
- Use the `INPUT_PROMPT` as the user request, prepending PR context (diff, title, description)
- For review specifically, restrict tools to read-only (no Write/Edit/Bash) since it's just reviewing

### Step 3: Add `direct_prompt` mode to prompt construction
Add a function in `create-prompt/index.ts` that builds the prompt from PR data + a direct user prompt, without extracting from a trigger comment.

### Step 4: Add example workflow file `.github/workflows/cortex-code-review.yml`
A ready-to-use review workflow triggered on `pull_request: [opened, synchronize]`.
